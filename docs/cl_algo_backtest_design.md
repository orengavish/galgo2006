# CL Algo Backtest System — Design Document
**Date:** 2026-07-04  
**Status:** Approved for implementation (autonomous session)

---

## 1. Purpose

Simulate every combination of algo type + bracket params over every day that has:
- Full TRADES tick data for the symbol
- Full BID_ASK tick data for the symbol
- At least one armed critical line in the DB for that day

Score each combination. Feed scores into a learning machine that narrows the parameter
search space across iterations. The output is the optimal (algo_type, params) combo for
live trading, derived from historical simulation rather than sparse verified trades.

---

## 2. Why Not Use Verified Trades as Primary Input?

Verified trades (real IB fills) are expensive — few per day, hard to accumulate.
bt_matrix_runner uses them to vary EXIT params around known entries.

This system is complementary: it takes CRITICAL LINES (freely available, entered daily)
and simulates BOTH ENTRY AND EXIT against tick data. No verified trades needed.
The more historical days with tick data + lines, the better the signal.

## 3. Cartesian Axes

| Axis | Values | Count | Rationale |
|------|--------|-------|-----------|
| `algo_type` | BOUNCE, BREAKOUT, DIRECTIONAL, FADE, BOTH | 5 | All strategies from algo_engine |
| `tp_ticks` | 2, 3, 4, 5, 6, 8, 10, 12 | 8 | Fine-grained in tight range, sparse beyond |
| `sl_ticks` | 2, 3, 4, 5, 6, 8, 10, 12 | 8 | Same |
| `direction_filter` | ALL, BUY_ONLY, SELL_ONLY | 3 | Test directional bias |
| `strength_max` | 1, 2, 3 | 3 | Strong-only vs all lines |

**Total: 5 × 8 × 8 × 3 × 3 = 2,880 combos per (day, symbol)**

Estimated throughput: ~5,000 simulations/second (pure Python + pandas).
With 10 days of history × 1 symbol × 10 avg lines/day × 2 avg commands/combo:
= 2,880 × 10 × 20 = 576,000 simulations → ~2 minutes.

## 4. Entry Simulation Model

Signal time: **RTH open = 8:30 CT** for all commands generated by algo_engine.
Commands are all "placed" at open; entry fills whenever the price condition is met.

### LMT Entry (BOUNCE, FADE, DIRECTIONAL with approaching price)
- Source: BID_ASK ticks (`bid_p`, `ask_p` columns)
- LMT BUY fills when: `ask_p ≤ entry_price` — fill at `entry_price` (R-BT-01)
- LMT SELL fills when: `bid_p ≥ entry_price` — fill at `entry_price` (R-BT-02)
- Scan from signal_time (8:30 CT) forward until session end

### STP Entry (BREAKOUT, DIRECTIONAL with retreating price)
- Source: TRADES ticks (`price` column)
- STP BUY fills when: `price ≥ entry_price` — fill at `entry_price + 1 tick` (slippage)
- STP SELL fills when: `price ≤ entry_price` — fill at `entry_price - 1 tick` (slippage)
- Same scan window

### Exit Simulation
- Delegates entirely to existing `simulator.simulate_exit()` (TRADES ticks, R-BT-04/05/06)
- OCO: SL checked before TP on same tick (R-BT-06)
- Session end (15:15 CT) → EXPIRED if no exit (R-BT-07)

### No Replenishment
Unlike live trading, simulation does NOT replenish after fills.
Each (line, direction) pair generates exactly one command per combo.
Rationale: replenishment complexity adds noise; one-pass is cleaner for scoring.
Add replenishment in a later iteration once base signal is clear.

## 5. "Iterate on Same Data" Mechanics

Every simulation result is stored with:
```
UNIQUE(date, symbol, algo_type, tp_ticks, sl_ticks, direction_filter, strength_max,
       line_price, direction)
```

`INSERT OR IGNORE` everywhere — re-running never duplicates rows.

**Iteration adds value by exploring NEW parameter values:**
- Run 1 (coarse): tp in [2,4,6,8,12], sl in [2,4,6,8,12] → 5×5=25 of the 64 tp/sl combos
- Learner finds hot zone: tp=4-6, sl=3-5
- Run 2 (fine): tp in [3,4,5,6,7], sl in [2,3,4,5,6] → NEW rows added for tp=3,5,7 etc.
- Same historical days, new param combinations → genuinely new data

Score history table tracks every scoring run → convergence detection.

## 6. Learning Machine Architecture

### Iterative Bayesian Grid Narrowing

```
Iteration 0: coarse full grid (tp/sl at [2,4,6,8,12])
     ↓
  Score → find top-20% combos in tp/sl space
     ↓
  Learner computes "hot zone" centroid in (tp,sl) space
  Generate finer grid: ±2 ticks around centroid at 1-tick resolution
  Keep 20% exploration budget: random from unexplored region
     ↓
Iteration 1: run backtester on new (tp,sl) pairs only
     ↓
  Score again → convergence check
     ↓
Iteration 2+: repeat until top-5 combos stable for 3 runs
     ↓
  CONVERGED: lock best combo, write to learner_runs as CONVERGED
```

algo_type, direction_filter, strength_max are categorical → always test all values,
just narrow the continuous (tp,sl) grid.

### Convergence Criterion
Top-5 combo IDs (by composite score) unchanged across 3 consecutive scoring runs.

### Sparse Data Guards (relaxed from plan_highlevel.md)
- Min fills gate: N ≥ 3 (not 20 — data is sparse)
- Stability zone: top combo must have at least 2 neighbors (±1 tp or ±1 sl) also positive PF
- Monte Carlo + LOOCV: deferred until N ≥ 30 fills for a combo

### Composite Score Formula
```
score = 0.30 × expectancy_norm    # mean pnl_ticks per trade (primary)
      + 0.25 × profit_factor_norm  # gross wins / gross losses
      + 0.20 × win_rate_norm       # % positive exits
      + 0.15 × sharpe_norm         # mean / std × sqrt(N)
      + 0.10 × sqn_norm            # Van Tharp SQN
```
All metrics min-max normalized across the scored combo set.
EXPIRED-only combos score = 0 regardless.

## 8. Concurrency Contract (Parallel Claude Instances)

```
Claude A: python cl_algo_worker.py --symbol MES
Claude B: python cl_algo_worker.py --symbol MNQ
```

- Shared DB: galao.db with `PRAGMA journal_mode=WAL; PRAGMA synchronous=NORMAL`
- Symbol partitioning: each worker writes only rows for its symbol
- Lock file per symbol: `data/cl_algo_{SYMBOL}.lock` — prevents duplicate workers
- `INSERT OR IGNORE` on all sim_results rows → crash-safe, restart-safe
- No shared mutable memory state
- Score + learner run per-symbol → no cross-symbol entanglement

## 10. Autonomous Decisions Made Today

| Decision | Rationale |
|----------|-----------|
| Signal time = 8:30 CT for all commands | Matches live decider behaviour |
| No replenishment in simulation | Reduces noise; add later when base signal clear |
| Min-N gate = 3, not 20 | Data is sparse; 20 is unreachable per combo at start |
| Stability zone = 2 neighbors | Relaxed from 8 (too strict for sparse grid) |
| Monte Carlo deferred to N≥30 | Not enough data today |
| STP entry uses TRADES+slippage | BID_ASK not guaranteed to show STP trigger moment |
| Initial coarse grid: tp/sl in [2,4,6,8,12] | 5 values → 25 tp/sl combos × 5 algos × 3 dirs × 3 strengths = 1,125 combos (fast first run) |
| Composite weights: expectancy 30%, PF 25% | Expectancy is the most actionable metric for futures |
